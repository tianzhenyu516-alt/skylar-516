***

## 一、项目背景

### 1.1 问题提出

- 工业生产中需要快速识别并分拣特定颜色物体
- 传统人工分拣效率低、成本高
- 需要自动化解决方案

### 1.2 项目目标

- 实现**实时视觉识别**黄色目标物体
- 通过**机械舵机**完成自动推走动作
- 与**STM32控制系统**协同工作
- 整个流程控制在**12秒内**完成

## 三、核心技术

### 3.1 视觉识别算法

#### 3.1.1 颜色空间选择

- **LAB颜色空间**优于RGB
- L：亮度通道
- A：绿-红通道（关键）
- B：蓝-黄通道（关键）

#### 3.1.2 黄色阈值定义

```python
yellow_threshold = [(50, 95, -15, 8, 35, 80)]
#       L: 50-95  亮度范围
#       A: -15~8  限制绿色，偏向红黄
#       B: 35-80  黄色强度
```

#### 3.1.3 多维度特征验证

1. **形状特征**：长宽比 > 1.2，过滤圆形干扰
2. **密度特征**：像素密度 > 0.25，排除稀疏噪点
3. **颜色验证**：中心点RGB颜色二次确认
   - 绿色占比 ≤ 8%
   - G比B高 40%以上

### 3.2 快速检测机制

- **3帧确认**：约0.05秒快速识别
- 状态机设计：SCAN → SIGNAL → PUSHING → DONE
- 非阻塞处理：识别与舵机控制并行

### 3.3 舵机控制

#### 3.3.1 舵机类型

- **连续旋转舵机** vs 角度舵机
- 1.0ms = 顺时针全速
- 1.5ms = 停止
- 2.0ms = 逆时针全速

#### 3.3.2 角度控制实现

```python
def servo_angle(angle):
    # 0-270度映射到500-2500us脉宽
    pulse_us = int(500 + (angle / 270.0) * 2000)
    duty = int(pulse_us / 20000.0 * 65535)
    servo.duty_u16(duty)
```

### 3.4 双机通信

- **GPIO4** 输出数字信号
- 高电平(3.3V) = 停止传送带
- 低电平(0V) = 恢复传送带
- **共地设计**确保信号可靠

## 四、系统流程

### 4.1 完整工作流程

```
┌─────────────────────────────────────────────────────┐
│  START: 系统初始化，舵机0度，GPIO4=LOW              │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  SCAN: 实时图像采集，3帧快速检测黄色目标            │
└──────────────────┬──────────────────────────────────┘
                   │ 检测到黄叶
                   ▼
┌─────────────────────────────────────────────────────┐
│  SIGNAL (0.05s)                                     │
│  • GPIO4 = HIGH (通知STM32停止传送带)               │
│  • 等待1秒让电机停稳                                │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  PUSHING (1.5s)                                     │
│  • 舵机从0度转到270度                               │
│  • 推走黄叶                                         │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  WAIT_RETURN (1s)                                   │
│  • 舵机转回0度                                      │
│  • 等待系统稳定                                     │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  DONE                                               │
│  • GPIO4 = LOW (通知STM32恢复传送带)                │
│  • 重置状态，准备下一次检测                         │
└─────────────────────────────────────────────────────┘
```

### 4.2 时间分配

| 阶段     | 时间         | 说明       |
| ------ | ---------- | -------- |
| 检测     | \~0.05s    | 3帧快速确认   |
| 等停电机   | 1s         | 传送带惯性停止  |
| 推走     | 1.5s       | 0°→270°  |
| 回位     | 1s         | 270°→0°  |
| 等恢复    | 1s         | 系统稳定     |
| **总计** | **\~4.5s** | 远小于15秒限制 |

## 五、技术难点与解决方案

### 5.1 难点一：强光干扰

**问题**：光线过强导致黄色泛白，识别困难

**解决**：

- LAB颜色空间对光照不敏感
- 动态调整阈值范围
- 中心点颜色二次验证

### 5.2 难点二：绿色误识别

**问题**：绿色物体被误判为黄色

**解决**：

- A通道严格限制（-15\~8）
- RGB颜色验证：绿色占比≤8%
- 多维度特征联合判定

### 5.3 难点三：实时性要求

**问题**：传送带运动，检测必须快

**解决**：

- 3帧快速确认机制
- 降低像素阈值，加速处理
- 非阻塞状态机设计

### 5.4 难点四：双机协同

**问题**：K230与STM32通信不可靠

**解决**：

- GPIO数字信号简单可靠
- 共地设计消除电平漂移
- 1秒延时确保电机停稳
